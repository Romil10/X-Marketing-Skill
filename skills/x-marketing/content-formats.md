# X Marketing — Content Formats Library

Reference for the x-marketing skill. Use these as starting structures, not templates to fill in robotically.

---

## Hook Types

Every post begins with a hook. Match the hook type to the content and the audience's current state.

| Hook Type | When to use | Pattern |
|---|---|---|
| **Curiosity** | You have a counterintuitive insight | "Most people think X. They're wrong." / "X is not what you think it is." |
| **Data** | You have a specific, surprising number | "[Specific number] [surprising implication]" |
| **Pain Point** | You know a specific frustration your audience feels | "[The exact problem they experience] — here's why it keeps happening." |
| **Opinion** | You have a defensible contrarian take | "Hot take: [contrarian but arguable position]" / "Unpopular opinion:" |
| **Story** | You have a real experience that proves a point | "3 months ago, [specific situation]. Here's what I learned:" |
| **Question** | You want to spark engagement + make them think | Ask a question they've never been asked but immediately care about |

**Rules:**
- Never open with a rhetorical question used as a trick to get clicks. Only use question hooks when the question is genuinely interesting.
- The hook must work as a standalone tweet. If no one clicks the thread, the hook still delivers value.
- Test: would you stop scrolling for this? If not, rewrite it.

## Tweet Formats

### Short Tweet (under 140 chars)
Best for: sharp insights, opinions, observations that need no explanation
```
[One sentence that lands on its own]
```
Example: "Data doesn't make decisions. Frameworks do."

### Value Tweet (140–280 chars)
Best for: tactical advice, quick how-tos, reframings
```
[Statement or claim]

[1–3 lines of supporting logic or context]

[Optional: one-line CTA or closer]
```

### List Tweet
Best for: toolkits, quick-reference points, checklists
```
[Hook: "X things that Y"]

• [Point 1]
• [Point 2]
• [Point 3]
• [Point 4]
• [Point 5]

[Closer: the meta-point or CTA]
```

### Success Tweet
Best for: milestones, proof points, social credibility
```
[The milestone — specific number or outcome, no hype]

[What it actually means in context]

[What made it possible — one real thing, not a list of gratitude]
```

### Failure Tweet
Best for: trust-building, relatability, teaching through experience
```
[What failed — honest and specific]

[Why it failed — the real reason, not the palatable one]

[What changed — the actual fix]
```

### Insight Tweet
Best for: original observations, frameworks, mental models
```
[The insight stated plainly]

[Why this is true / what it explains]

[The implication — what changes if you believe this]
```

### Quote Tweet
Best for: building off trending conversations, adding perspective to others' posts
```
[Affirm, challenge, or extend the original post in 1–2 sentences]

[Add the specific angle or data point they missed]
```

## Thread Formats

Threads are for depth. Every thread must:
- Have a hook tweet that stands alone (gets value even if they stop after tweet 1)
- Advance clearly — each tweet should follow logically from the last
- End with a closer that callbacks to the hook and includes one clear CTA

### Educational Thread
Structure: Hook → Problem → Solution (3–5 tweets) → Proof or Example → Closer + CTA
```
1/ [Hook: the counterintuitive claim or the promise]

2/ [The problem: why this matters and why the default approach fails]

3/ [Step 1 of the solution]

4/ [Step 2 of the solution]

5/ [Step 3 of the solution]

6/ [Real example or proof point]

7/ [Closer: restate the core insight, CTA — follow / reply / save]
```

### Story Thread
Structure: Hook (the ending or the stakes) → Beginning → Rising Action → Turning Point → Lesson → CTA
```
1/ [Open with the most interesting moment — not the beginning]

2/ [Back up: how it started]

3-5/ [The journey — specific details, no generalizations]

6/ [The turning point or moment of realization]

7/ [The lesson — stated plainly]

8/ [Closer: what this means for the reader]
```

### Framework Thread
Structure: Hook → Why this framework exists → Each component (1 tweet each) → How to apply it → CTA
```
1/ [Hook: "Here's the [X]-part framework for [outcome]"]

2/ [Why existing approaches fail and why this framework works]

3/ [Component 1: name + one line explanation]

4/ [Component 2]

5/ [Component 3]

6/ [How to apply the full framework — a real example]

7/ [Closer: the core trade-off or insight the framework reveals]
```

### Tool List Thread
Structure: Hook (the problem this list solves) → Tools (1–2 per tweet) → Summary + CTA
```
1/ [Hook: "X tools I use for [outcome] — most people don't know half of these"]

2-5/ [2–3 tools per tweet: name + what it does + one specific use case]

6/ [Closer: how they work together / which to start with / CTA]
```

### Case Study Thread
Structure: Hook (the result) → Context → What they did → Why it worked → What you can take from it
```
1/ [Hook: the surprising outcome]

2/ [Who this is about + the situation they were in]

3-5/ [What they did — specific actions, not generalities]

6/ [Why it worked — the real reason]

7/ [What you can take from it — the transferable principle]
```

## Reply Strategy Formats

### Value Reply
Add a specific point, data point, or reframe to a conversation. Never just agree.
```
[Affirm briefly if warranted] + [Add the specific thing they missed or extend their point]
```

### Early Reply (get in early on viral posts)
Reply within the first 30 minutes on posts from accounts in your niche.
Focus: add genuine insight, not just visibility. The reply should be worth reading on its own.

### Conversation Reply
When someone replies to your post — respond to every reply in the first hour.
Keep replies substantive. A one-word response signals low effort. One sentence minimum.

## Content Mix Guidelines

A healthy weekly X schedule (adjust based on capacity):

| Format | Frequency | Purpose |
|---|---|---|
| Value/Educational tweet | 2–3/week | Authority + reach |
| Opinion or hot take | 1/week | Engagement + identity |
| Thread | 1/week | Deep authority + profile growth |
| Engagement post (poll, question, quote tweet) | 1/week | Community |
| Personal/Story tweet | 1/week | Trust + human connection |

**Rule:** No more than 2 consecutive posts of the same type or pillar. Vary the feed.

**Posting times (general defaults — adjust based on your analytics):**
- Best windows: 7–9am, 12–1pm, 6–8pm in your audience's primary timezone
- Threads: Tuesday–Thursday, morning
- Engagement posts: Friday or weekend

## Platform Specs

| Element | Spec |
|---|---|
| Tweet character limit | 280 |
| Thread optimal length | 4–8 tweets |
| Hook tweet | Must work as a standalone post |
| Hashtags | 0–2 max, only when genuinely topical. Never use trending hashtags for reach. |
| Links | X suppresses reach on posts with external links in the tweet body. Use reply-to-own-tweet for links, or "link in bio" framing. |
| Images | Native images get better reach than link previews. Use for charts, screenshots, carousels. |
| Polls | Use sparingly — once every 2–3 weeks. Must be a genuine question, not a vanity engagement trick. |
